# Docs Index

이 문서는 문서들의 역할과 단일 소스를 빠르게 찾기 위한 인덱스입니다.

## 빠른 링크
- `README.md`: 프로젝트 개요/시작 방법
- `AGENTS.md`: 기여자 가이드/작업 규칙
- `docs/codebase-analysis.md`: 운영/개발 컨텍스트, 코드베이스 요약, 리스크
- `docs/frontend.md`: 프론트엔드 상세 동작/UX 스펙
- `docs/backend.md`: 백엔드 구현 계획 (API 계약은 plan/api.md가 소스)
- `docs/plan/prd.md`: 제품 요구사항 (PRD)
- `docs/plan/api.md`: API 계약 단일 소스

## 문서 역할 원칙
- `docs/codebase-analysis.md`: 요약 + 리스크만 유지
- 상세 동작/스펙은 해당 가이드 문서에서 관리
  - 프론트: `docs/frontend.md`
  - 백엔드: `docs/backend.md`
- 요구사항/계약 변경은 plan 문서를 우선 반영
  - `docs/plan/prd.md`, `docs/plan/api.md`
