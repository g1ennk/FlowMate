## ✨ 최근 개선 사항 (2024-01-09)

### 1. Break에서 정지(■) 버튼 ✅
**변경 내용**: Phase별 다른 Toast 메시지
- **Flow**: "기록됨" (시간 기록됨)
- **Break**: "타이머 종료" (기록 없음을 명확히)

**이유**: 사용자가 기록 여부를 명확히 알 수 있도록

---

### 2. Break에서 완료(✓) 버튼 ✅
**변경 내용**: 비활성화 (Flow에서만 완료 가능)
- `disabled={!isFlow}`
- Tooltip: "Flow 중에만 완료 가능"

**이유**: 휴식 중에 태스크를 완료하는 것은 논리적으로 맞지 않음

---

### 3. 수동 스킵(→) 시 기록 ✅
**변경 내용**: 기록 안함
- 스킵 = "넘어가기"의 의미
- 기록하려면 정지(■) 버튼 사용

**이유**: 스킵과 정지의 명확한 구분

---

### 4. Phase별 닫기 확인 ✅
**변경 내용**: Phase별 다른 확인 메시지
- **Flow**: "Flow가 진행 중입니다..."
- **Break**: "휴식이 진행 중입니다..."
- **Waiting**: 확인 없이 바로 닫기

**이유**: 사용자가 현재 상태를 명확히 인지

---

### 5. Waiting 상태 닫기 ✅
**변경 내용**: 확인 모달 없이 바로 닫기
- Waiting은 아직 시작하지 않은 상태이므로 확인 불필요

---

### 6. 정지 후 상태 유지 및 재개 ⭐ 최신
**변경 내용**: 정지(■) 시 pause 상태로 저장하여 재개 가능
- 일반 타이머: 정지 → 닫기 → 재오픈 시 이전 시간에서 이어서 진행
- 뽀모도로: 정지 → 닫기 → 재오픈 시 이전 phase/시간/cycle에서 이어서 진행
- pause 상태에서 닫기 시 확인 모달 없음
- sessionStorage에 타이머 상태 저장

**이유**: 작업을 중단하고 나중에 이어서 할 수 있는 유연한 UX 제공

### 💡 정지 후 재개 핵심 포인트

1. **정지(■) = pause + 닫기**
   - 타이머 상태는 sessionStorage에 저장
   - 완전히 종료(stop)가 아님

   - pause 상태면 바로 해당 타이머 화면으로
   - 사용자 의도: "이어서 하기"

3. **모드 변경하고 싶다면?**
   - 리셋(↻) → 닫기 → 재오픈 → 모드 선택
   - 또는 완료(✓) 처리 후 새로 시작
